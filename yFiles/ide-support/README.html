yFiles for HTML IDE Support

IDE Support

The files in this directory can be used to enhance the development experience while developing yFiles for HTML applications.

yFiles Developer Tools

These tools provide type checking for library members at runtime and API information in your browser's JavaScript console. You can enable it by including the JavaScript file yfiles-typeinfo.js.

For detailed descriptions see the Developer's Guide section yFiles Developer Tools.

TypeScript Type Definitions

yFiles comes with TypeScript type definitions for both the UMD and the ES module version of the yFiles for HTML library. These files can be used to integrate yFiles for HTML into your TypeScript project, and, for JavaScript projects, to enable code completion, parameter info, code inspections, and quick documentation lookup for the yFiles API in IDEs that support it.

See the Developer's Guide section Setting up Your Development Environment for more information.

The following TypeScript tye definition files are included:

yfiles-api-umd-webstorm.d.ts
Typings for the UMD module version of the library, using HTML in the source code documentation. Recommended for WebStorm users.
yfiles-api-es-modules-webstorm.d.ts
Typings for the ES module version of the library, using HTML in the source code documentation. Recommended for WebStorm users.
yfiles-api-umd-vscode.d.ts
Typings for the UMD module version of the library, using markdown in the source code documentation. Recommended for Visual Studio Code users.
yfiles-api-es-modules-vscode.d.ts
Typings for the ES module version of the library, using markdown in the source code documentation. Recommended for Visual Studio Code users.

JavaScript/JSDoc Type Definitions

Similar to a TypeScript type definitions, yFiles comes with type definitions in JavaScript/JSDoc format. This can be used to enable code completion, parameter info, code inspections, and quick documentation lookup for the yFiles API in IDEs that don't support the TypeScript type definitions for JavaScript projects. Many modern IDEs, especially WebStorm, IntelliJ IDEA, and Visual Studio Code, prefer TypeScript type definitions even for JavaScript projects because of TypeScripts more expressive type system.

The JavaScript/JSDoc type definitions are provided by the file yfiles-api-umd-jsdoc.js.

See the Developer's Guide section Setting up Your Development Environment for more information.
